# Plan 2.2: Auto-Populate PLAN.md Template in createPlan

## Objective
Extend `createPlan` so that when `projectRoot` is provided but no explicit `content` is given, it auto-loads `.gsd/templates/PLAN.md`, populates it with plan variables (phaseNumber, planNumber, wave, name), and writes the result to the filesystem.

This completes the GSD template integration for all three entity types.

## Context
- src/core/plan.ts
- src/core/template_gsd.ts
- src/core/content.ts
- .gsd/templates/PLAN.md

## Tasks

<task type="auto">
  <name>Auto-populate PLAN.md template in createPlan when no content provided</name>
  <files>
    - src/core/plan.ts
  </files>
  <action>
    In `createPlan`, after the DB insert, modify the filesystem-write block:

    Current behaviour:
    - Only writes if `content !== undefined && projectRoot`

    New behaviour:
    - If `content !== undefined && projectRoot` → write `content` as before (explicit content wins)
    - Else if `content === undefined && projectRoot` → attempt template auto-populate:
      1. Import `loadGsdTemplate`, `populatePlanTemplate` from `./template_gsd.js`
      2. Look up `phase.number` and `phase.milestone_id` from DB (already done in existing code path)
      3. Call `loadGsdTemplate(projectRoot, 'PLAN.md')`
      4. If found, call `populatePlanTemplate(raw, { phaseNumber: phase.number, planNumber: number, wave: wave ?? 1, name })`
      5. Call `writePlanContent(projectRoot, phase.milestone_id, phase.number, number, populated)`
      6. If template not found, silently skip.

    Do NOT change behaviour when `projectRoot` is not provided.

    Note: The existing early-exit `if (content !== undefined && projectRoot)` block needs to be refactored into an `if (projectRoot)` block that branches on whether `content` was provided.
  </action>
